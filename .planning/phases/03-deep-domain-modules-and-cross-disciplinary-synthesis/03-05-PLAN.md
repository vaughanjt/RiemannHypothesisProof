Create `src/riemann/analysis/conjecture_gen.py` with:

1. `ExperimentSuggestion` dataclass:
```python
@dataclass
class ExperimentSuggestion:
    type: str          # "spectral" | "trace" | "modular" | "padic" | "tda" | "dynamics" | "ncg" | "analogy" | "cross_domain"
    description: str   # Human-readable description of the experiment
    rationale: str     # Why this experiment is suggested
    parameters: dict   # Suggested parameter values
    priority: float    # 0.0-1.0, higher = more important
```

2. `suggest_experiments(db_path: str | Path | None = None, max_suggestions: int = 5) -> list[ExperimentSuggestion]`:
   - Read workbench state: list_experiments(db_path), list_conjectures(db_path=db_path)
   - Strategy A (empty workbench): return bootstrap suggestions covering all 8 domains:
     - "Run Berry-Keating spectral analysis at N=200" (spectral, priority=0.9)
     - "Compute persistence diagrams of zero embeddings" (tda, priority=0.8)
     - "Scan Bost-Connes phase transition near beta=1" (ncg, priority=0.7)
     - "Test spectral-zeros analogy mapping" (analogy, priority=0.85)
     - "Compute Weil explicit formula convergence" (trace, priority=0.75)
   - Strategy B (has experiments): analyze existing experiments:
     - Count experiments by type (from parameters.get("type", "unknown"))
     - Under-explored domains get higher priority
     - Speculative conjectures without supporting experiments get "test this conjecture" suggestions
     - Anomalies (conjectures tagged "anomaly") get "investigate this anomaly" suggestions
   - Sort by priority descending, truncate to max_suggestions
   - Return list of ExperimentSuggestion

3. `analyze_results(experiment_id: str, db_path: str | Path | None = None) -> dict`:
   - Load experiment via load_experiment
   - Raise ValueError if experiment not found
   - Parse experiment parameters and result_summary
   - Produce structured analysis:
     - "summary": Brief 1-sentence summary of what the experiment found
     - "patterns_detected": List of pattern strings extracted from result_summary (split on periods/semicolons, filter non-empty)
     - "anomalies": List of anomaly-like findings (result_summary substrings containing "deviat", "unexpect", "anomal", "surpris")
     - "suggested_conjectures": List of dicts {"statement": str, "confidence": float} derived from patterns
     - "next_experiments": List of dicts {"type": str, "description": str} for follow-up work
   - Return the analysis dict

4. `generate_conjecture(observations: list[str], evidence_ids: list[str] | None = None, confidence: float = 0.0, db_path: str | Path | None = None) -> str`:
   - Synthesize observations into a formal conjecture statement
   - Statement format: "Observed: {'; '.join(observations)}. Conjecture: [pattern holds generally]"
   - Use create_conjecture with evidence_level=0, status="speculative", confidence=confidence, tags=["ai_generated", "computational_evidence"]
   - If evidence_ids provided: link each experiment to the conjecture via link_evidence with relationship="supports"
   - Return conjecture_id

Write TDD tests using temp_db fixture. Test empty workbench bootstrap, test with pre-populated experiments, test analyze_results parsing, test generate_conjecture persistence.
